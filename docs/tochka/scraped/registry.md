# Реестры

Реестры отправляются на почту, которую вы указали при онбординге. Если нужно изменить её, напишите в чат в интернет-банке или менеджеру сопровождения.

Формат реестра — zip-архив, который содержит csv-файл. В качестве разделителей в csv-файле используется точка с запятой, разделитель дробной части чисел — запятая. Кодировка файла — UTF-8.

## Процессинговый реестр операций

Эти реестры отправляются ежедневно. Если операций в предыдущий календарь не было, придёт пустой файл.

Формат реестра:

| Название колонки | Значение |
| --- | --- |
| TRANS\_DATE | Дата создания операции |
| TRANSACTION\_ID | Идентификатор транзакции (`PaymentUid` или `RefundUid`) |
| SUM | Сумма операции |
| COMMISSION | Сумма комиссии по операции |
| ITV\_COMMISSION | Сумма комиссии ИТВ |
| USER\_INFO | Маскированный номер карты |
| MERCH | Идентификатор мерчанта |
| MERCH\_SITE | Идентификатор сайта |
| PARENT\_TRANSACTION\_ID | Для возвратов указывается `PaymentUid` исходной операции платежа, для платежей поле остаётся пустым |
| TRANSACTION\_TYPE | Тип транзакции (`PAYMENT` или `REFUND`) |
| MERCHANT\_SITE\_NAME | URL сайта мерчанта |
| PAYMENT\_METHOD\_TYPE | Метод оплаты (`CARD` или `SBP`) |
| RRN | RRN карточной операции (платежа или возврата) |
| AUTH\_CODE | Код авторизации для карточной операции |
| ORDER\_UID | Уникальный идентификатор заказа в информационной системе ТСП |
| QRC\_ID | Идентификатор зарегистрированной функциональной ссылки СБП |
| COMMISSION\_NDS | Сумма НДС по комиссии за операцию |
| ITV\_COMMISSION\_NDS | Сумма НДС по комиссии за ИТВ |

## Реестр чарджбеков

Реестры чарджбеков отправляются вручную при наличии чарджбеков минимум два раза в неделю.
